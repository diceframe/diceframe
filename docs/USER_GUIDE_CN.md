# DiceFrame 用户手册

这份文档给 GM 和玩家看。它只讲怎么开局、怎么行动、怎么掷骰、怎么用群聊 Bot，不讲代码结构。

## 第一次打开

启动 WebUI 后，浏览器打开终端里显示的地址，默认是：

```text
http://localhost:18000
```

第一次进入设置页，先填模型配置：

- API 地址：兼容 OpenAI Chat Completions 的接口地址。
- 模型名：例如 `deepseek-v4-pro`。
- API Key：模型服务商提供的密钥。

保存后可以用设置页里的测试连接确认模型能正常回复。

## 开第一局

1. 进入“创建”。
2. 选择游戏语言。
3. 选择模板世界、AI 生成世界，或自己填写世界设定。
4. 选择规则和难度。
5. 创建角色，或先让 AI 生成草稿再手动修改。
6. 进入“游玩”。
7. 输入角色行动，例如“我检查墙上的符文”。

DiceFrame 会把玩家行动交给 GM 模型，模型返回叙事和状态标签。系统会解析这些标签，更新 HP、物品、金币、经验、场景等状态。

## 语言和内容

“设置”里的界面语言只影响 WebUI 文案；“创建”里的游戏语言会写入存档，用来约束 GM 叙事、开场、摘要、快捷行动和 AI 生成内容。

世界模板、世界书和内容包有自己的内容语言。创建页会优先列出同语言模板，也会标出其他模板或世界书的内容语言。选择中文世界书时，英文界面仍会显示这本世界书的中文内容；选择英文世界书时，中文界面也不会把原文自动改写成中文。

规则按语言拆分文件：中文版 `<rule_id>.json` + 英文版 `<rule_id>_en.json`。创建游戏时按游戏语言自动加载对应版本，缺少英文版时用中文版。

## 回合和行动

单人模式里，提交行动后通常会直接推进。

多人模式里，每轮每个活跃玩家都可以提交自己的行动。所有活跃玩家都提交后，系统会推进到下一段 GM 叙事。GM 也可以强制推进，适合有人暂时不在、或当前回合只需要部分玩家行动的时候。

玩家可以修改本轮行动，但有次数上限。AI 只读取最后一次提交的行动。

## 多人玩法

GM 创建游戏后，可以复制邀请链接发给其他玩家。

玩家打开链接后可以：

- 加入当前游戏。
- 创建角色。
- 认领已有角色。
- 在自己的玩家页面提交行动。

GM 可以在游戏页查看玩家状态。有人临时不在时，可以标记“暂离”；暂离玩家不会阻塞回合，剧情里默认跟随队伍，不主动做重大决定。回来后再恢复即可。

## 掷骰流程

有些行动会直接叙事，有些行动需要检定。

当系统判断行动需要骰子时，会先停下来提示掷骰。玩家或 GM 完成掷骰后，再继续推进叙事。这样做是为了保留跑团的仪式感，也避免 GM 模型在不知道骰点的情况下直接写结果。

常见流程是：

1. 玩家提交行动。
2. 系统提示这次行动需要检定。
3. 玩家点击或发送掷骰。
4. 骰点写入当前行动。
5. GM 根据骰点继续叙事。

如果玩家不方便处理，GM 可以代为推进；多人局里也可以由 GM 强制推进。

## 状态变动怎么看

GM 叙事后，系统会显示“状态变动”。这里记录的是这轮真正写入存档的变化。

常见状态包括：

- HP：受到伤害或治疗。
- 物品：获得、失去或消耗道具。
- 金币：获得、支付或扣除。
- 经验：获得经验、升级。
- 场景：当前位置或当前局势变化。
- 私密感知：只给某个角色看的信息。

如果状态变动和叙事文字不一致，以状态变动为准；叙事文字是描述，状态变动是系统实际记录。

## 群聊 Bot

DiceFrame 可以通过 QQ / NapCat 插件把网页对局接到群聊里。Bot 只通过 Web 服务的 HTTP API 工作，不直接读写存档。

基本流程：

1. 在 WebUI 设置页打开插件配置。
2. 配置 NapCat 的 WebSocket 地址、端口和 token。
3. 启用 `QQ / NapCat` 插件。
4. 在游戏页复制 Bot 绑定命令。
5. 到群里发送绑定命令。

绑定成功后，群聊就和当前网页游戏关联起来了。

## 群聊常用命令

下面用 `@bot` 表示艾特你的 Bot。

```text
@bot 帮助
@bot 加入 角色名
@bot 新建角色
@bot 车卡
@bot AI车卡
@bot 邀请
@bot 前情
@bot 地图
@bot 状态
@bot 感知
@bot 支付
@bot 掷骰
@bot 我检查墙上的符文
@bot 推进
@bot 下一轮
@bot 暂离
@bot 回来
```

常用说明：

- `加入 角色名`：把你的群聊账号和网页里的角色绑定。
- `新建角色` / `车卡`：获取建卡入口或建卡提示。
- `AI车卡`：让 AI 生成角色草稿，确认后再使用。
- `邀请`：发送玩家加入链接。
- `前情`：查看公开剧情提要和最近回合。
- `地图`：查看当前地点和已知地点关系。
- `状态`：查看自己的角色摘要。
- `感知`：查看只属于自己的私密信息，通常会私聊发送。
- `支付`：查看待确认付款，并可确认或拒绝。
- `掷骰`：确认当前待处理检定。
- `推进` / `下一轮`：GM 或授权账号推进剧情。
- `暂离` / `回来`：临时离开或恢复参与。

## 群聊行动

玩家可以直接在群里艾特 Bot 发送自然语言行动：

```text
@bot 我绕到守卫背后，观察他腰间钥匙的位置
```

如果行动需要检定，Bot 会提示发送 `@bot 掷骰`。如果不需要检定，或骰子已经处理完，系统会继续推进并把 GM 叙事和状态变动发回群里。

## 私密信息

有些信息只给特定角色看，例如暗门、幻觉、梦境、内心低语、角色独有线索等。

网页左侧会显示角色感知。群聊里可以发送：

```text
@bot 感知
```

Bot 会尽量私聊发送；如果私聊失败，会提示检查临时会话或好友设置。

## 常见问题

### AI 没回复

先检查设置页的模型配置和测试连接。常见原因是 API Key 错误、模型名错误、接口地址不兼容，或网络无法访问模型服务。

### 玩家不能行动

可能是角色已死亡、还没有加入本局、当前回合正在推进，或者玩家只是 GM 预览身份。刷新页面前先看页面提示。

### Bot 没反应

先确认 WebUI 里的 QQ / NapCat 插件已启用，再确认 NapCat WebSocket 地址、端口和 token 正确。群聊还需要先完成绑定。

### 群聊里没有推进

多人局可能还在等待其他活跃玩家提交行动。GM 可以发送 `@bot 推进` 或在网页里强制推进。

### 状态看起来不对

先刷新游戏详情。如果叙事文字和状态变动冲突，以状态变动为准。仍然异常时，优先保留当前存档，不要删除 `data/`，方便排查。

### 可以公开存档或群聊记录吗

不建议。`data/` 里可能包含 API Key、访问口令、真实群号、私密消息和完整跑团记录。发布到 GitHub 前不要提交 `data/`、`.env`、日志和缓存。
