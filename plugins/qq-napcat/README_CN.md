# QQ / NapCat 插件

1. 在 NapCat 进入“网络设置 → 网络配置 → WebSocket 服务器”，启用服务并记下端口和 access_token。
2. 在 DiceFrame 设置的插件页填写连接配置并启用。
3. 在房主页复制 Bot 绑定命令并发送到目标群聊。
4. 玩家发送 `@Bot 加入 角色名` 后可用自然语言行动。

常用命令：

- `@Bot 帮助`：查看当前群可用指令。
- `@Bot 绑定 <game_key> <一次性凭证>`：GM 将当前网页游戏绑定到群聊，凭证成功后立即作废。
- `@Bot 邀请`：发送当前局网页加入链接，并附一张新玩家一图流教程卡。
- `@Bot 邀请 @玩家` / `@Bot 邀请我`：在群里保留公开链接，同时额外私聊指定玩家或自己；如果 Bot 无法发起私聊，会在群里提示添加好友/开启临时会话，并让玩家使用公开链接兜底。
- `@Bot 新建角色` / `@Bot 车卡`：在群里发送车卡填写建议和建卡入口，不依赖私聊。
- `@Bot AI车卡`：若开启 AI 辅助车卡，Bot 才会尝试私聊玩家收集自然语言描述；生成草稿后由玩家确认并发到群里公示。
- `@Bot 加入 角色名`：玩家认领网页中已有角色。
- `@Bot 前情`：查看公开前情提要和最近回合。
- `@Bot 地图`：查看当前场景和世界书地点连接，未认领角色也可用。
- `@Bot 状态`：查看已认领角色的 HP、金币、背包等摘要。
- `@Bot 感知`：私聊发送最近的角色专属感知。
- `@Bot 支付`：私聊发送待确认支付列表。
- `@Bot 确认支付` / `@Bot 拒绝支付`：处理待确认付款。
- `@Bot 掷骰`：确认需要骰子的行动。
- `@Bot 推进` / `@Bot 下一轮`：由 GM 或设置中授权的账号强制推进当前回合。
- `@Bot 暂离` / `@Bot 回来`：临时下线时不阻塞回合；GM 可发送 `@Bot 暂离 角色名` / `@Bot 让角色名回来` 指定角色。
- `@Bot <自然语言行动>`：提交角色行动；若这条行动触发 GM 生成，Bot 会先提示“GM 正在思考中”。

没有公网网页入口时，`邀请` / `新建角色` / `车卡` / `地图` 仍会给出群聊内可读的说明或卡片；配置了网页公开地址时，会附带可点击网页链接。

AI 辅助车卡默认开启，但只有玩家明确发送 `@Bot AI车卡` 才会尝试私聊；普通 `新建角色/车卡` 全在群里闭环。AI 车卡只调用独立角色生成接口，不进入正式剧情上下文，不写入回合日志，也不会自动创建角色；玩家私聊确认后仅把公开版草稿发到群里，GM 如需调整可去网页角色页修改。

暂离角色仍在队伍中，但不会计入待行动人数；AI 上下文会标记其“默认跟随队伍，不主动做重大决定”。玩家回来后发送 `@Bot 回来` 即可重新参与。

推进权限默认只有绑定本局的 GM；如需让副 GM 或可信玩家协助推进，可在插件配置的“可推进的账号 ID”里每行填写一个账号。NapCat 下通常填写 QQ 号。

图片卡片缓存：

- Bot 会把帮助、状态、车卡教程等图片临时保存到 `data/bot/cards`。
- 插件配置里可以设置“卡片缓存保留时长”和“最多保留张数”，发送新卡片时会自动清理旧的 `card_*.png`。
- 设置页提供“清理卡片缓存”按钮，可立即删除这些临时 PNG；不会删除游戏存档、角色卡或其他图片。
